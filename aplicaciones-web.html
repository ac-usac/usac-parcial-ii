Estructura de las Aplicaciones Web

Las aplicaciones web son programas que se ejecutan en un servidor y se acceden a través de un navegador web. Se dividen fundamentalmente en dos partes que trabajan en conjunto: el Frontend (lado del cliente) y el Backend (lado del servidor).

a. Frontend (Lado del Cliente)

Definición

Es la parte de la aplicación con la que el usuario interactúa directamente. Incluye todo lo que ve en el navegador: la estructura, el diseño, los colores, las imágenes y cualquier interacción de interfaz de usuario (UI). Es el responsable de la **experiencia de usuario (UX)**.

b. Tecnologías Clave del Frontend

1. HTML (HyperText Markup Language)

Es el lenguaje de marcado fundamental. Define la **estructura** y el **contenido** de la web (títulos, párrafos, enlaces, imágenes, formularios). Versión clave para el examen: **HTML5**, que introduce etiquetas semánticas (`

`, `
`) y soporte nativo para multimedia.

2. CSS (Cascading Style Sheets)

Se encarga de la **presentación** y el **diseño**. Controla la apariencia visual (colores, tipografía, disposición, animaciones). **Concepto vital:** la **cascada** determina qué reglas de estilo se aplican a un elemento. Se utiliza junto con frameworks como Bootstrap.

3. JavaScript (JS)

Añade **interactividad** y **comportamiento dinámico**. Permite modificar el DOM (Modelo de Objetos del Documento) en tiempo real, manejar eventos, validar formularios y realizar llamadas asíncronas al backend (AJAX/Fetch). Es el lenguaje de programación del navegador.

c. Backend (Lado del Servidor)

Definición y Funcionalidad

Es la lógica de la aplicación que reside en un servidor remoto. Es invisible para el usuario, pero crucial. Se encarga de:

  • Lógica de Negocio: Procesar datos, realizar cálculos y ejecutar las reglas de la aplicación.
  • Autenticación y Autorización: Verificar quién es el usuario y a qué recursos tiene acceso.
  • Interacción con la Base de Datos: Almacenar, recuperar y modificar la información persistente.

d. Componentes y Tecnologías del Backend

1. Base de Datos (DB)

El sistema de almacenamiento de datos de la aplicación. Puede ser **Relacional (SQL)** como MySQL o PostgreSQL (ideal para datos estructurados), o **NoSQL** como MongoDB o Redis (ideal para datos flexibles o caché).

2. Servidor (Web Server)

Es el software que escucha peticiones HTTP (ej. Apache, Nginx) y las dirige a la lógica de la aplicación. El Servidor de Aplicaciones (Application Server) ejecuta el código del backend, generando la respuesta a enviar al cliente.

3. Lenguajes de Programación

Lenguajes utilizados para escribir la lógica de negocio. Ejemplos comunes incluyen: **Python** (Django, Flask), **Java** (Spring), **Node.js** (JavaScript/Express), **PHP** (Laravel, Symfony) y **Ruby** (Ruby on Rails).

e. Desarrollo Fullstack

Definición

Un desarrollador o pila (stack) Fullstack maneja y comprende todas las capas de una aplicación web: desde la configuración del servidor y la gestión de bases de datos (Backend), hasta la interfaz de usuario interactiva y estilizada (Frontend). El término es crucial en la práctica moderna, ya que muchas empresas buscan perfiles que puedan cubrir ambos lados del desarrollo.
