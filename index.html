Desarrollo de Software: Guía Introductoria

a. Proyecto de Software

1. Concepto y Características

Un proyecto de software es un esfuerzo único, temporal y planificado, dirigido a crear un producto o servicio de software específico. Se caracteriza por tener un inicio y fin definidos, objetivos claros, y la necesidad de gestión de recursos y riesgos.

2. Alcances y Secuencia

El **Alcance** define qué se incluirá (funcionalidades) y qué no, limitando el trabajo. La **Secuencia** (o cronograma) organiza las tareas en fases lógicas (planificación, diseño, codificación) para garantizar la entrega ordenada.

3. Ejemplos

  • Desarrollo de una app móvil para reservas.
  • Implementación de un sistema ERP.
  • Creación de un módulo de seguridad en un portal web.
  • Actualización de un sistema operativo.

b. Tipos de Proyectos de Software

La clasificación de proyectos depende del entorno o plataforma donde el software será ejecutado:

  • 1. Aplicaciones Web Navegador
  • 2. Aplicaciones Móviles iOS/Android
  • 3. Aplicaciones para Escritorio PC/Mac/Linux
  • 4. Sistemas Embebidos Hardware Específico
  • 5. Desarrollo de Videojuegos Entretenimiento
  • 6. Otro (Ej. Sistemas de I.A.) Aprendizaje/Datos

c. Ciclo de Vida del Software y sus Fases

El SDLC (Software Development Life Cycle) es una metodología para producir software de alta calidad y bajo costo.

Fase inicial donde se define el objetivo del proyecto, se establece el alcance, se estiman los costos y se elabora el cronograma.

Se recopilan y documentan los requisitos del sistema, especificando qué debe hacer el software para satisfacer las necesidades del usuario.

Se crea la arquitectura del sistema, la interfaz de usuario, y se definen los módulos, bases de datos y algoritmos a utilizar.

Se escribe el código fuente del software basándose en las especificaciones de la fase de diseño.

Se realizan tests (unitarios, integración, sistema) para identificar y corregir errores, asegurando que el software cumple con los requisitos.

El software se instala y se pone en funcionamiento en el entorno real de los usuarios.

Actividades post-implementación para corregir fallos, mejorar el rendimiento y adaptar el software a cambios en el entorno o requisitos.
